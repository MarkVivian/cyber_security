                SCP
            ============
- This is a secure copy utility for securely transferring files between hosts on a network.
- Uses SSH (secure shell) to provide encryption.
- Common options:
    -r: Recursively copy entire directories.
    -P port: Specify the SSH port to use (capital P).
    -p: Preserves modification times, access times, and modes from the original file.
    -q: Disables the progress meter.
    -C: Enables compression.

                
                USAGES.
            --------------
1. copy a file from a local system to a remote system.
    EG.
        scp ./local_file username@remote_host:path/to/destination

            - To handle filenames with spaces.
                EG.
                    scp "./local file with spaces.txt" username@remote_host:"/path/to/destination/"
                            OR 
                    scp "user@host:'/path/to/remote file with spaces.txt'" ./localdir/

            - Or use rsync -e ssh with --protect-args (safer for arbitrary filenames)
                EG.
                    rsync -e ssh --protect-args ./local\ file\ with\ spaces.txt username@remote_host:/path/to/destination/

                        > copies local file with spaces.txt to remote destination preserving spaces.
                        > 

2. copy a file from remote system to a local system
    EG.
        scp username@remote_host:path/to/file ./local_machine


3. copy a file between two remote systems
    EG.
        scp username1@remotehost1:/path/to/remotefile username2@remotehost2:/path/to/destination


4. copy a directory recursively
    EG.
        scp -r localdir username@remotehost:/path/to/destination


5. specify the port 
    EG.
        scp -P 2220 username@remotehost:/file ./local/
        

6. preserve file attributes
    EG.
        scp -p localfile username@remotehost:/path/to/destination

             > -p preserves modification times, access times and modes from the original file.


7. modern alternative:
    > sftp (secure file transfer protocol) for interactive file transfers.
        * Interactive or batch SFTP client (sftp user@host), safer and more featureful than scp for transfers.

    > rsync -e ssh (rsync over SSH) for syncing files and directories.
        * great for efficient syncs (only transfers diffs), preserves metadata, supports resume.
        * More robust and flexible than scp for large or repeated transfers.

        EG.
            # sftp interactive session.
            sftp user@host 

            # rsync over ssh
            rsync -avz -e ssh /local/dir/ user@host:/remote/dir/
                OR 
            rsync -avz -e "ssh -p 2222" /local/dir/ user@host:/remote/dir/
